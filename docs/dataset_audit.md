## 4. Ground Truth for Localization

| Metric | Value |
|--------|-------|
| GT map entries (step2_gt_map.json) | 299 |
| Seaport GT entries (seaport_gt.json) | 466 |
| PTX positive bags with gt_idx | 292 |
| **Interior GT bags** (gt_idx not at last position) | **101** |
| Last-only GT bags (excluded from localization eval) | 191 |

**Interior GT bags** are the 101 bags where the ground truth transaction is not the final transaction in the bag. These are the only bags used for localization evaluation, as they provide a non-trivial test of the model's ability to identify suspicious transactions without relying on recency bias.

**Relabeling protocol:** For contract-mediated phishing (ice_phishing, payable_function), the labeled hash in PTXPhish is the user-facing approval transaction. We trace ERC-20 `transferFrom` and internal ETH transfers in execution receipts to identify the true fraudulent transaction. For Seaport-based NFT phishing (nft_order), we parse `OrderFulfilled` events to identify the settlement transaction. This relabeling affects 9.3% of cases.
